Portfolio Reflection – Artemis Financial Security Project
Project Summary and Client Requirements

Artemis Financial is a consulting company that creates individualized financial plans for customers. Because the company handles sensitive financial and personal data, its main requirement was secure software communication and protection against cyber threats. The goal of this project was to identify vulnerabilities in its RESTful web application and recommend solutions to improve security.

Security Vulnerabilities and Secure Coding

I performed well in identifying vulnerabilities through manual review and static testing. Key issues included missing input validation, lack of authentication, hard-coded credentials, and weak error handling. Secure coding is important because it prevents data breaches, protects customer trust, and supports regulatory compliance. Strong security adds value by reducing risk and improving system reliability.

Challenges and Helpful Learning

The most challenging part was analyzing how small coding decisions could create larger security risks. However, this process helped me better understand real-world software security and how structured assessments can reveal weaknesses in APIs, encryption, and data handling.

Increasing Security Layers and Future Assessments

I improved security by recommending input validation, authentication controls, encrypted database connections, better encapsulation, and dependency updates. In the future, I would continue using tools like OWASP Dependency-Check along with threat modeling, secure coding standards, and automated testing to assess vulnerabilities and choose mitigation strategies.

Ensuring Functionality and Security

After refactoring, I verified that the application remained functional by testing the software and reviewing code behavior. I also reran security checks to confirm that new vulnerabilities were not introduced during changes.

Tools and Practices Used

Helpful tools and practices included OWASP Dependency-Check, manual code review, secure coding principles, and vulnerability assessment frameworks. These resources helped me identify risks and apply practical security improvements that I can use in future projects.

Value for Future Employers

This project demonstrates my ability to perform vulnerability assessments, identify security risks, and recommend mitigation strategies. I would show this artifact to employers as evidence of my secure coding knowledge, problem-solving skills, and ability to document technical findings clearly.
